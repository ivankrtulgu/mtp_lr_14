```
Требования:

2. Внедрить механизм буферизации:

```

**Результат:** Модифицирован сборщик данных. Внедрена стратегия Timed-Batching с использованием `time.Ticker` и внутреннего слайса-буфера. Это позволило сократить количество системных вызовов записи в файл, объединяя по 100 транзакций в одну операцию записи. Верификация через логи подтвердила корректную работу пакетного сброса.

## Задание 3: Реализация graceful shutdown

### Промпт 1
**Инструмент:** Sisyphus $\to$ golang-pro (Gemma 4 31B)
**Промпт:**
```
Ты — senior Go разработчик. Добавь в сборщик данных механизм Graceful Shutdown для корректного завершения работы при получении системных сигналов (SIGINT, SIGTERM).

Требования:
1. Использовать context.Context и signal.NotifyContext для отслеживания сигналов завершения.
2. Реализовать координированный останов конвейера:
   - Reader должен прекратить чтение CSV-файла при отмене контекста.
   - Воркеры должны дообработать все строки, уже попавшие в канал linesChan.
   - Writer должен выполнить финальный сброс (flush) всех данных из буфера и из канала resultsChan перед выходом.
3. Обеспечить синхронизацию: основная функция main не должна завершаться до того, как Writer подтвердит полную очистку буферов и закрытие файлов.

Результат: обновленный файл main.go с внедренным механизмом обработки сигналов и гарантией сохранности данных при прерывании.
```

**Результат:** Внедрен механизм Graceful Shutdown. Теперь при нажатии Ctrl+C программа не завершается мгновенно, а последовательно останавливает этапы конвейера, дорабатывает остатки данных и корректно закрывает файл. Это исключает потерю записей, находившихся в памяти в момент сигнала.

### Итого
- **Количество промптов:** 1
- **Что пришлось исправлять вручную:** Провел серию ручных тестов Graceful Shutdown, прерывая программу через Ctrl+C в разные моменты выполнения. Верифицировал через логи последовательность остановки компонентов (Reader $\rightarrow$ Workers $\rightarrow$ Writer) и подтвердил, что финальный сброс буфера данных происходит корректно без потерь записей.
